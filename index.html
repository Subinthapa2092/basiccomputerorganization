4. MapReduce & Word Count

MapReduce is a programming model for processing large datasets in parallel across a cluster of computers. It has two phases: Map and Reduce.

💡 Key Difference from Traditional Programming Traditional: Sequential processing on single machine. MapReduce: Parallel distributed processing across many nodes. Can handle petabytes of data efficiently.
PAST Q2nd Assessment 2081
What is MapReduce? How does it work? Explain the word counting process of MapReduce. [1+1+4]
MapReduce = A parallel data processing framework with 2 phases:
• Map phase: Splits input → emits (key, value) pairs. e.g., "hello" → (hello, 1)
• Shuffle & Sort: Groups by key. e.g., hello → [1, 1]
• Reduce phase: Aggregates values per key. e.g., hello → 2

5. Hadoop & its Components

Hadoop is an open-source framework for storing and processing big data in a distributed environment using commodity hardware.

⭐

HDFS Key Concept: NameNode manages file metadata (directory structure). DataNodes store actual data blocks (default 128MB each). Data is replicated 3 times for fault tolerance.

6. Types of Machine Learning

AI vs Machine Learning

  • AI = Broader concept (machines simulate intelligence)
  • ML = Subset of AI (machines learn from data)
  • All ML is AI, not all AI is ML
  • AI includes robotics, expert systems etc.

ML vs Traditional Programming

  • Traditional: Input + Rules → Output
  • ML: Input + Output → Rules (learned)
  • Traditional is explicit, ML is learned
  • ML handles complex patterns automatically

7. Regression Techniques

Linear Regression

Predicts a continuous output variable based on one input variable. The relationship is modeled as a straight line.

y = mx + c    OR    y = β₀ + β₁x where: y = predicted output, x = input, m/β₁ = slope, c/β₀ = intercept

Polynomial Regression

Used when the relationship is non-linear. Fits a curved line through the data points.

y = β₀ + β₁x + β₂x² + β₃x³ + ... + βₙxⁿ

Multiple Linear Regression

Uses multiple input variables to predict one output.

y = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + ... + βₙxₙ e.g. House price = β₀ + β₁(area) + β₂(rooms) + β₃(location)

8. Classification Algorithms

KNN — K Nearest Neighbors

KNN classifies a new point by finding its K nearest neighbors and taking a majority vote of their classes.

📐 Euclidean Distance Formula d(p,q) = √[(x₂-x₁)² + (y₂-y₁)²]
For n dimensions: d = √[Σ(qᵢ - pᵢ)²]

Naive Bayes Classifier

Uses Bayes' theorem assuming all features are independent. Great for text classification.

P(Class | Features) = P(Features | Class) × P(Class) / P(Features) Simplified: P(Y|X) ∝ P(X|Y) × P(Y) P(Yes|features) = P(f1|Yes) × P(f2|Yes) × P(f3|Yes) × P(Yes)

Decision Tree

Tree-like model where each internal node = feature test, each leaf = class label. Easy to interpret.

🔴 KEY COMPARISON: Logistic Regression vs Linear Regression Linear Regression → Continuous output (e.g. price). Logistic Regression → Binary output (Yes/No, 0/1). Logistic uses sigmoid function to squash output to [0,1].

9. Clustering — K-Means & K-Medoids

K-Means Algorithm

⚙ K-MEANS STEPS (EXAM STANDARD)
1
Choose K (number of clusters). Randomly select K centroids.
2
Assign each data point to the nearest centroid using Euclidean distance.
3
Recalculate centroids as the mean of all points in each cluster.
4
Repeat steps 2-3 until centroids don't change (convergence).

K-Means

  • Centroid = Mean of cluster points
  • Faster, works well on large data
  • Sensitive to outliers
  • Centroid may not be a real data point

K-Medoids

  • Medoid = Actual data point (representative)
  • More robust to outliers & noise
  • Slower, more computationally expensive
  • Centroid is always a real data point

10. Model Evaluation — Confusion Matrix

✅ RMSE & MAE — For Regression Evaluation MAE = Mean Absolute Error = (1/n) Σ|yᵢ - ŷᵢ| (average absolute error)
RMSE = Root Mean Squared Error = √[(1/n) Σ(yᵢ - ŷᵢ)²] (penalizes large errors more)

2ND ASSESSMENT

Q: What is machine learning? How is it different from traditional programming?

Machine Learning is a branch of AI where systems learn from data and improve their performance over time without being explicitly programmed.

Differences:
• Traditional: Programmer writes rules → Computer follows rules → Output
• ML: Data + Output (examples) → Computer learns rules → Can predict new outputs
• Traditional fails on complex patterns (e.g., recognizing faces). ML handles this naturally.
• Traditional: deterministic. ML: probabilistic.
• Example: Spam filter — Traditional needs manual rules. ML learns patterns from 1000s of emails.
2ND ASSESSMENT

Q: How do you differentiate database and data warehouse? Explain.

Database: Used for day-to-day operations (OLTP). Stores current data. Fast insert/update/delete. Small to medium size. Examples: MySQL, PostgreSQL.

Data Warehouse: Used for analytics and reporting (OLAP). Stores historical data. Complex queries/aggregations. Very large size. Requires ETL process. Examples: Amazon Redshift, Snowflake.

Key point: Data Warehouse collects data FROM multiple databases and transforms it for analysis.

Q: What is data visualization? Why is it used? Explain scatterplot and histogram with its use case. [2+4]

Data Visualization = Graphical representation of data using visual elements like charts, graphs, and maps to understand trends, patterns, and outliers.
Why used: Makes data understandable · Reveals hidden patterns · Supports decision making · Communicates insights quickly

Scatter Plot: Shows relationship/correlation between two numerical variables. Each point = one observation. Use case: Studying relationship between study hours and exam scores. Can show positive/negative/no correlation.

Histogram: Shows frequency distribution of a single numerical variable. Bars represent ranges (bins) of values. Use case: Understanding age distribution of customers, showing whether data is normally distributed.
1ST ASSESSMENT 2081

Q: What is confusion matrix? Write formula for accuracy, precision and recall.

Confusion Matrix = A table used to evaluate classification model performance. It shows TP, TN, FP, FN counts.
TP = Correctly predicted positive | TN = Correctly predicted negative
FP = Predicted positive but actually negative (Type I Error)
FN = Predicted negative but actually positive (Type II Error)

Accuracy = (TP + TN) / (TP + TN + FP + FN) → Overall correct predictions
Precision = TP / (TP + FP) → Of all predicted positive, how many were correct?
Recall = TP / (TP + FN) → Of all actual positives, how many did we catch?
F1 Score = 2 × (Precision × Recall) / (Precision + Recall) → Harmonic mean
BOARD EXAM 2081

Q: What is cognitive bias? Explain anchoring bias, sampling bias, fundamental attribution error and stereotyping. [2+4]

Cognitive Bias = Systematic error in thinking that affects judgments and decisions.

Anchoring Bias: Over-relying on the first piece of information encountered. e.g. A product marked "Was $100, Now $60" — $100 anchors our perception of value.

Sampling Bias: When sample is not representative of the population. e.g. Surveying only urban people about internet usage.

Fundamental Attribution Error: Tendency to attribute others' behavior to character rather than situation. e.g. "He failed because he's lazy" (ignoring difficult circumstances).

Stereotyping: Applying a generalized belief about a group to an individual. e.g. Assuming someone from a country is good at math because their nationality is stereotyped that way.

Numerical 1 — K-Means Clustering

🔴 EXAM QUESTION (Both Assessments & Board Exam) Divide {(2,10), (2,5), (8,4), (5,8), (7,5), (6,4)} into 2 clusters using K-Means.
🔢 SOLUTION — K-MEANS (K=2)
1
Initialize: Randomly pick K=2 centroids. Let C1 = (2,10), C2 = (8,4)
2
Calculate distances from each point to C1(2,10) and C2(8,4):
d(p, C1) = √[(x-2)² + (y-10)²] | d(p, C2) = √[(x-8)² + (y-4)²] (2,10): d1=0, d2=√[36+36]=8.49 → Cluster 1 (2,5): d1=5, d2=√[36+1]=6.08 → Cluster 1 (8,4): d1=√[36+36]=8.49, d2=0 → Cluster 2 (5,8): d1=√[9+4]=3.61, d2=√[9+16]=5.00 → Cluster 1 (7,5): d1=√[25+25]=7.07, d2=√[1+1]=1.41 → Cluster 2 (6,4): d1=√[16+36]=7.21, d2=√[4+0]=2.00 → Cluster 2
3
Cluster 1: {(2,10), (2,5), (5,8)} | Cluster 2: {(8,4), (7,5), (6,4)}
4
Recalculate Centroids:
New C1 = ((2+2+5)/3, (10+5+8)/3) = (3, 7.67) New C2 = ((8+7+6)/3, (4+5+4)/3) = (7, 4.33)
5
Repeat until centroids don't change. Final Result:
✅ Cluster 1: {(2,10), (2,5), (5,8)} — Centroid ≈ (3, 7.67)
✅ Cluster 2: {(8,4), (7,5), (6,4)} — Centroid ≈ (7, 4.33)

Numerical 2 — KNN Classification

🔴 EXAM QUESTION (2nd Assessment 2081) Class A: (2,2),(2,3),(3,2) | Class B: (4,5),(5,3),(6,2). Classify point (4,3). K=3, Euclidean.
🔢 SOLUTION — KNN (K=3)
1
Calculate Euclidean distance from (4,3) to ALL points:
d = √[(x₂-x₁)² + (y₂-y₁)²] Class A: (2,2): √[(4-2)²+(3-2)²] = √[4+1] = √5 ≈ 2.24 (2,3): √[(4-2)²+(3-3)²] = √[4+0] = 2.00 (3,2): √[(4-3)²+(3-2)²] = √[1+1] = √2 ≈ 1.41 Class B: (4,5): √[(4-4)²+(3-5)²] = √[0+4] = 2.00 (5,3): √[(4-5)²+(3-3)²] = √[1+0] = 1.00 (6,2): √[(4-6)²+(3-2)²] = √[4+1] = √5 ≈ 2.24
2
Sort by distance (ascending) and pick K=3 nearest:
1st: (5,3) Class B — distance 1.00 ✓ 2nd: (3,2) Class A — distance 1.41 ✓ 3rd: (2,3) Class A — distance 2.00 ✓ (tie with (4,5) — pick both or either)
3
Majority Vote: Class A = 2 votes, Class B = 1 vote
✅ Result: (4,3) is classified as Class A

Numerical 3 — Naive Bayes Classifier

🔴 EXAM QUESTION (Board Exam 2081) Using the 11-row dataset, classify: Color=Yellow, Origin=Imported, Type=Formal using Naive Bayes.
🔢 SOLUTION — NAIVE BAYES
1
Prior Probabilities (from dataset — 11 rows, Yes=6, No=5):
P(Yes) = 6/11 | P(No) = 5/11
2
Likelihood — Count occurrences in dataset:
For Yes class (6 records): P(Yellow|Yes) = count(Yellow AND Yes)/count(Yes) P(Imported|Yes) = count(Imported AND Yes)/count(Yes) P(Formal|Yes) = count(Formal AND Yes)/count(Yes) For No class (5 records): P(Yellow|No), P(Imported|No), P(Formal|No) — similarly counted
3
Posterior (unnormalized):
P(Yes|features) ∝ P(Yellow|Yes) × P(Imported|Yes) × P(Formal|Yes) × P(Yes) P(No|features) ∝ P(Yellow|No) × P(Imported|No) × P(Formal|No) × P(No) The class with HIGHER value is the prediction.
4
✅ Classify to whichever P(class|features) is greater.
Always show all calculations step by step for full marks!
