Memory Trick

Fetch-Decode-Execute Cycle

STEP 1 — FETCH:
  MAR ← PC           (copy next address)
  MDR ← Memory[MAR]  (read instruction)
  IR  ← MDR          (load into IR)
  PC  ← PC + 1       (advance to next)

STEP 2 — DECODE:
  Control Unit examines IR
  Identifies operation + operands

STEP 3 — EXECUTE:
  ALU performs the operation
  Result stored in Accumulator/register
  PSW flags updated

STEP 4 — STORE (if needed):
  Result written back to memory
  
  → Repeat from Step 1 for next instruction

Pipelining overlaps execution of multiple instructions — like an assembly line. Instead of finishing one instruction before starting the next, different stages process different instructions simultaneously.

WITHOUT Pipeline:  I1[IF ID AG EX WB] I2[IF ID AG EX WB]  = 10 cycles

  Clock:  1   2   3   4   5   6   7

Pipeline Hazards

Data Hazard: Next instruction needs result of current one. Control Hazard: Branch disrupts pipeline. Structural Hazard: Two instructions need same hardware resource.

📋 Past Exam Questions — Chapter 2
Q1 HOT ×3
What is Pipelining? How it helps in maximizing performance? Explain 5-stage Pentium pipeline.
Pipelining is a technique where multiple instructions are overlapped in execution like an assembly line. Without pipelining: one instruction completes before next starts. With pipelining: different stages process different instructions simultaneously → increases throughput. 5-stage Pentium: IF (Instruction Fetch) → ID (Instruction Decode) → AG (Address Generation) → EX (Execute) → WB (Write Back). Problems: Data hazard, Control hazard, Structural hazard.
Q2 HOT ×3
Compare and contrast RISC and CISC architecture.
RISC: few simple instructions, fixed length, 1 clock cycle, many registers, easy to pipeline, less power, used in ARM/mobile. CISC: many complex instructions, variable length, multiple cycles, few registers, hard to pipeline, more power, used in Intel x86/desktop. RISC puts more work on compiler; CISC puts more work on hardware.
Q3 MEDIUM
What is memory? Explain memory hierarchy of a digital computer.
Memory stores data and instructions. Hierarchy (fastest to slowest): Registers (bytes, inside CPU) → Cache L1/L2/L3 (KB–MB) → RAM/Main Memory (GB) → HDD/SSD (TB) → Tape/Optical. Going up: faster, smaller, expensive. Going down: slower, larger, cheap. CPU checks registers first, then cache, then RAM, then disk.
Q4
What is byte ordering? Explain Little-Endian and Big-Endian format.
Byte ordering defines how multi-byte data is stored in memory. Big-Endian: Most Significant Byte at lowest address (natural order) — used in internet/Motorola. Little-Endian: Least Significant Byte at lowest address (reversed) — used in Intel x86/AMD. Example: 0x12345678 stored as 12 34 56 78 (Big) or 78 56 34 12 (Little).

Universal Gates

NAND and NOR are called Universal Gates because ANY logic gate (AND, OR, NOT) can be built using ONLY NAND gates or ONLY NOR gates. This reduces hardware complexity in chip design.

De Morgan's Theorems

Theorem 1:  (A · B)' = A' + B'
            NOT(A AND B) = NOT(A) OR NOT(B)

Theorem 2:  (A + B)' = A' · B'
            NOT(A OR B) = NOT(A) AND NOT(B)

Proof of Theorem 1:
A  B │ A·B │ (A·B)' │ A' │ B' │ A'+B'
0  0 │  0  │   1    │  1 │  1 │   1   ← Equal ✓
0  1 │  0  │   1    │  1 │  0 │   1   ← Equal ✓
1  0 │  0  │   1    │  0 │  1 │   1   ← Equal ✓
1  1 │  1  │   0    │  0 │  0 │   0   ← Equal ✓

Role of Data Path

The data path physically moves and transforms data. The Control Unit tells the data path WHAT to do (via control signals). Together they implement the instruction set of the CPU.

Microinstructions & Notation

Each machine-level instruction is broken down into a sequence of microinstructions — simple, primitive operations that directly control hardware components of the data path.

NotationMeaningExample
MAR ← PCCopy PC value to MAR registerLoad address for fetch
MDR ← M[MAR]Read from memory at address MAR into MDRFetch instruction
IR ← MDRCopy MDR contents to IRLoad instruction
PC ← PC + 1Increment Program CounterAdvance to next instruction
AC ← AC + MDRALU adds AC and MDR, result to ACAddition operation
M[MAR] ← MDRWrite MDR contents to memory at MARStore operation

Microinstruction Execution for LOAD instruction

Machine Instruction: LOAD R1, 1000  (load memory[1000] into R1)

Microinstruction Sequence:
Cycle 1: MAR ← PC           (get instruction address)
Cycle 2: MDR ← M[MAR],      (fetch instruction from memory)
         PC  ← PC + 1
Cycle 3: IR  ← MDR           (decode: it's a LOAD)
Cycle 4: MAR ← IR[address]   (get operand address = 1000)
Cycle 5: MDR ← M[MAR]        (read data from memory[1000])
Cycle 6: R1  ← MDR           (store data in R1)

Introduction to Pipelining

Pipelining at the microarchitecture level divides instruction execution into stages, allowing the data path to work on multiple instructions simultaneously — greatly improving throughput.

Without Pipeline (sequential):
─────────────────────────────
I1: [Fetch][Decode][Execute]
I2:                          [Fetch][Decode][Execute]
Time: 6 units

With Pipeline (overlapped):
────────────────────────────
Clock:  1       2       3       4       5
I1:   [Fetch][Decode][Execute]
I2:           [Fetch][Decode][Execute]
I3:                   [Fetch][Decode][Execute]
Time: 5 units for 3 instructions!

How Control Signals Work

Microinstruction format contains control bits:
┌─────────┬──────────┬──────────┬──────────┬─────────┐
│ ALU_OP  │  SRC_REG │  DST_REG │ MEM_READ │  MEM_WR │
│ (4 bits)│ (3 bits) │ (3 bits) │  (1 bit) │ (1 bit) │
└─────────┴──────────┴──────────┴──────────┴─────────┘
Each bit/field directly controls a hardware component in the data path.

Unit 5 · 10 Hours

Instruction Set Architecture:
ISA, Addressing & Instructions

What is ISA?

The Instruction Set Architecture (ISA) is the interface between hardware and software — the set of all instructions a CPU can understand and execute.

Definition

ISA defines: the instruction set, data types, registers, memory model, addressing modes, and interrupt/exception handling. It is the contract between software writers and hardware designers.

Properties of Good ISA

PropertyMeaning
CompletenessMust support all necessary operations for any program
OrthogonalityInstructions should be independent — any instruction works with any addressing mode
RegularityConsistent and uniform — no special cases or exceptions
CompactnessInstructions encoded efficiently — small program size
EfficiencyFrequently used operations should be fast and short
Backward CompatibilityNew processors must run old programs — crucial for user adoption
PortabilityPrograms should run on different hardware implementations

Backward Compatibility

Why It Matters

Backward compatibility means new versions of a processor can run programs written for older versions without modification. Intel x86 maintains compatibility from 1978 to today. Without it, every software upgrade would require rewriting all existing programs — enormous cost and effort.

It is a critical factor in ISA design because: (1) billions of dollars of existing software investment must be protected, (2) users will not adopt a new architecture if it breaks existing programs, (3) it allows gradual hardware improvement while preserving software ecosystem.

Types of Instructions (by Operands)

THREE-ADDRESS instruction:   ADD R1, R2, R3     → R1 = R2 + R3
TWO-ADDRESS instruction:     ADD R1, R2         → R1 = R1 + R2
ONE-ADDRESS instruction:     ADD 1000           → AC = AC + Memory[1000]
ZERO-ADDRESS instruction:    ADD                → TOS = TOS + (TOS-1)  [stack machine]

Advantages:
3-address: Most flexible, fewer instructions needed
2-address: Most common — balance of flexibility and code size
1-address: Uses accumulator — simple hardware
0-address: Stack-based — very compact code (Java bytecode)

Memory Trick

Immediate = value IS here. Direct = address is here. Register = check register. Indirect = register has the address. Indexed = base + offset.

Memory Model & Registers in ISA

Memory Model defines how memory is organized and accessed — flat vs segmented, byte vs word addressing, endianness. It affects how programs store and retrieve data.

Registers are critical because: (1) they are the fastest accessible storage, (2) more registers means fewer memory accesses, (3) register allocation affects compiler efficiency, (4) the number and type of registers defines programming style and performance.

Both are important ISA design factors because they directly impact program performance, compiler design, and hardware complexity.

Virtual Memory & Paging

Virtual Memory

Virtual memory is a memory management technique that gives processes the illusion of having more memory than physically available by using disk storage as an extension of RAM.

Core Idea

Programs use logical (virtual) addresses. The OS and hardware translate these to physical addresses in actual RAM. Parts of the program not currently needed are stored on disk.

Paging

Paging divides logical memory into fixed-size blocks called pages and physical memory into blocks of same size called frames.

      │

┌──────────────┬───────────────┐
│ Frame Number │  Page Offset  │

└──────────────┴───────────────┘

Physical Address = Frame Number × Frame Size + Offset

Logical to Physical Address Translation

Step 1: CPU generates logical address (page number p, offset d)
Step 2: MMU looks up page number p in Page Table
Step 3: Page Table gives frame number f
Step 4: Physical address = f × page_size + d
Step 5: Access M[physical address]

Example:
  Page size = 1KB = 1024 bytes
  Logical address = 2050
  Page number p = 2050 ÷ 1024 = 2
  Offset d = 2050 mod 1024 = 2
  Page Table: Page 2 → Frame 5
  Physical address = 5 × 1024 + 2 = 5122

Page Table

A page table is maintained by the OS for each process. It maps page numbers to frame numbers. Each entry in the page table (PTE) contains:

FieldPurpose
Frame NumberPhysical frame where this page is stored
Valid/Present Bit1 = page is in RAM, 0 = page is on disk
Dirty Bit1 = page has been modified, needs to be written to disk
Reference Bit1 = page was recently accessed (for replacement)
Protection BitsRead/Write/Execute permissions
Modified BitIndicates whether the page contents have changed

Page Fault & Replacement Algorithms

Page Fault

A page fault occurs when a process tries to access a page that is NOT currently in RAM (Valid bit = 0). The OS must load the page from disk into RAM. If RAM is full, a page must be replaced.

Page Replacement Algorithms

AlgorithmStrategyNotes
FIFOReplace the page that has been in memory the longestSimple but not always optimal
LRUReplace the Least Recently Used pageGood performance, needs tracking
OptimalReplace page that won't be used for longest timeBest but impossible in practice
LFUReplace Least Frequently Used pageUses frequency counters
Clock/Second ChanceFIFO with second chance if reference bit = 1Approximates LRU efficiently

Segmentation

Segmentation divides memory into variable-size segments based on logical units (code, data, stack). Unlike paging (fixed size), segments match the logical structure of programs.

Logical Address: (Segment number, Offset)
Segment Table:
┌───────┬────────┬────────┐
│  Seg# │  Base  │  Limit │
├───────┼────────┼────────┤
│   0   │  1000  │   500  │  Code segment
│   1   │  2000  │  1000  │  Data segment
│   2   │  5000  │   200  │  Stack segment

Physical Address = Base + Offset (if Offset ≤ Limit)

Segmentation vs Paging:
Paging: fixed size blocks, no external fragmentation
Segmentation: variable size, matches logical structure, external fragmentation

What is page table? What information is stored in each page table entry?
Page table maps logical page numbers to physical frame numbers, maintained by OS per process. Each Page Table Entry (PTE) contains: Frame Number (physical location), Valid/Present bit (1=in RAM, 0=on disk), Dirty bit (page modified?), Reference bit (recently accessed?), Protection bits (read/write/execute permissions). Q3 HOT ×2
Explain the process of logical to physical address translation using page table.
1. CPU generates logical address (page number p, offset d). 2. MMU looks up page number p in page table. 3. If valid bit=1: get frame number f. If valid bit=0: PAGE FAULT — load from disk. 4. Physical address = frame number × page size + offset d. 5. Access data at physical address. Example: page size=1024, logical=2050 → p=2, d=2 → frame=5 → physical=5122.

Page fault: accessing a page not in RAM (valid bit=0), OS loads it from disk. If RAM full, must replace a page.
